# Status

This docs site is intentionally small and incremental.

It documents the current repo state, not the final design.

## Stable enough to document now

- project layout
- current CLI commands
- current scene syntax
- current script syntax
- inline scene-local script handlers/functions
- current ASCII map syntax
- maintained examples

## Still evolving quickly

- script language design
- reusable function model
- runtime query APIs like input/time
- Apple export flow
- procedural terrain direction

## Documentation policy

The intended workflow is:

- implement features in the repo
- document the current syntax here
- expand incrementally as the language/runtime stabilizes

This keeps the docs useful without pretending the language is already frozen.
