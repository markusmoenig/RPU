# RPU

RPU is a CLI-first runtime for interactive 2D scenes.

The current stack in this repo is:

- `rpu` as the top-level CLI
- `rpu-core` for project loading, scene parsing, map parsing, and script compilation
- `rpu-runtime` for the live runtime world and script execution
- `rpu-scenevm` for native windowing and rendering

RPU is currently focused on:

- declarative scene files
- lightweight scripts compiled to bytecode
- texture-backed 2D rendering
- embedded ASCII maps inside scene files
- hot reload during `rpu run`

## Current state

The language and runtime are still evolving.

What exists today:

- `rpu.toml` project manifest
- `scene`, `camera`, `rect`, `sprite`, `map`, `legend`, and `ascii`
- scripts with:
  - assignments
  - numeric expressions
  - locals via `let`
  - `if` / `else`
  - boolean conditions with `&&`, `||`, `!`
  - top-level reusable functions with `call name()`

What is intentionally not stable yet:

- the full script language design
- long-term map/terrain semantics
- Apple export flow details
- any future 3D direction

Use these docs as documentation of the repo's current behavior, not as a frozen language spec.
